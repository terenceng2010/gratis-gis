fix(web-map): keep icon-size constant, drop diagnostic logging 
 
MapLibre rejects feature-state expressions in layout properties 
(paint-only restriction). My stateCase-wrapped icon-size made 
the whole symbol layer fail to add, which is why icons vanished 
entirely in icon mode. Icon size is now a plain constant; the 
selection halo circle already provides the picked-feature visual 
via its paint-property expressions, which is where feature-state 
is actually allowed. 
 
Also removes the diagnostic console.log added in the previous 
commit now that the root cause is known.
