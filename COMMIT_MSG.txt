feat(web-map): selection state, map highlight, attribute table sync 
 
Selection is now first-class. Picks in the attribute table 
immediately highlight on the map, picks persist across layer 
switches inside the table, and a new per-layer Selectable toggle 
in Interactions opts a layer out of selection entirely. 
 
- Schema: interactions.selectable, default true. 
- map-editor owns selection as Record<layerId, Set<featureId>>. 
- Canvas mirrors selection to MapLibre feature-state (diffed, so 
  only changed ids touch the style) and paints selected features 
  with an accent outline, bumped radius/width, or a halo circle 
  under icon markers. 
- Attribute table reads/writes the shared selection instead of 
  its own local Set, so map and table stay in sync. 
- Layer panel gains a Selectable checkbox alongside Highlight on 
  hover. 
 
Next: click / rectangle / lasso / polygon draw tools that write 
into this same selection state.
