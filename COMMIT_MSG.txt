fix(web-map): disable SDF variant so icons always render 
 
Registering the plain raster with sdf: true made MapLibre render 
the point invisibly (alpha channel is binary, not a true distance 
field). Skip SDF registration for now; the canvas already falls 
back to the plain image whenever the SDF variant is missing, so 
icons render reliably in their shipped color. 
 
Tint toggle in the style editor remains so user preference 
persists, with a note that tinting awaits a proper SDF pipeline.
